# SmartDigit Lab｜手写数字识别工程演示网站

## 项目简介

SmartDigit Lab 是一个面向理工场景的手写数字识别 Web 演示项目。用户可以在网页画板中手写 0–9 的数字，或上传数字图片，系统会对输入图像进行预处理、特征提取与分类推理，并在页面右侧展示预测结果、置信度分布和诊断说明。

本项目用于展示人工智能在图像识别任务中的完整应用流程，包括数据预处理、模型推理、结果可视化、错误分析与迭代优化。项目可作为深度学习课程任务、AI 应用开发实践或工程演示网站使用。

## 核心功能

- 在线手写数字画板：支持用户直接在网页中书写 0–9 数字。
- 图片上传识别：支持上传本地数字图片进行识别演示。
- 预测结果展示：输出最终识别数字和置信度。
- 概率分布可视化：展示 0–9 各类别的预测概率。
- 输入预处理：对手写输入进行裁剪、归一化、居中和灰度化处理。
- 多策略融合识别：结合轻量模型、模板匹配、结构特征和反馈样例校正，提高识别稳定性。
- 诊断说明：根据笔迹特征给出识别稳定性和改进建议。

## 技术路线

项目整体流程如下：

```text
用户输入 / 图片上传
        ↓
图像裁剪与灰度化
        ↓
尺寸归一化至 28×28
        ↓
重心居中与特征提取
        ↓
轻量 MLP + 模板匹配 + 结构规则融合
        ↓
输出预测结果与置信度分布
        ↓
根据反馈持续迭代优化
```

## 项目特色

1. **前端轻量化部署**  
   项目采用纯静态网页实现，打开 `index.html` 即可运行，不依赖复杂服务器环境。

2. **可解释的识别过程**  
   页面不仅显示最终结果，还显示各类别置信度，便于分析模型为何出现误判。

3. **反馈驱动的迭代优化**  
   项目根据多轮真实手写样例反馈，对 `3/5`、`4/9`、`6/5`、`7/9`、`8/1` 等易混类别进行了针对性校正。

4. **便于课程提交和公开访问**  
   项目可直接部署到 GitHub Pages、Netlify 或 Vercel，生成公网 URL 后即可提交。

## 本地运行方法

1. 下载并解压项目文件夹。
2. 双击打开 `index.html`。
3. 在画板中手写数字，点击“开始识别”。
4. 查看预测结果和各类别置信度。

## 部署方法

### GitHub Pages 部署

1. 在 GitHub 新建公开仓库，例如 `smart-digit-lab`。
2. 上传 `index.html`、`README.md` 等项目文件。
3. 进入仓库的 `Settings` → `Pages`。
4. Source 选择 `Deploy from a branch`。
5. Branch 选择 `main`，目录选择 `/root`。
6. 保存后等待 GitHub 生成访问链接。

访问链接格式通常为：

```text
https://你的用户名.github.io/smart-digit-lab/
```

### Netlify 部署

1. 登录 Netlify。
2. 选择手动部署或拖拽部署。
3. 上传包含 `index.html` 的整个网站文件夹。
4. 部署完成后复制生成的公网链接。
